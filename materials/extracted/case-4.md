案例库 4：非地理性相关案例

本案例库聚焦「非地理性共享空间」议题：当空间失去地理意义时，如何建立「公共感」与「共识」？收录关注「虚拟空间」「数字社交」「共感系统」「共享记忆」等方向的案例，为非地理性的公共空间与共识系统设计提供参考。

**NS-01｜Thales Alenia Space “GATEWAY4U” 协作虚拟环境**

说明：由 Thales Alenia Space 开发，面向太空舱室设计阶段，构建协作式虚拟环境，供工程师与航天员在 XR（扩展现实）中共同体验与评估舱室的架构、布局、功能。研究显示，位于不同地点的参与者可“进入”同一虚拟舱室进行讨论、设计调整与反馈。

启发：体现“非地理性共享空间”：多人从各自位置进入同一虚拟舱室。；支持“动态生成／调整”：可实时更改布局与功能，模拟不同情境。；为在轨、可移动、位置变化的舱室提供 XR 协作与共识形成的空间模型。

技术载体：• 沉浸式技术（Immersive Technology）：使用 VR/MR 头显、空间定位追踪与实时渲染技术重建航天环境。；• 感应技术（Sensing Technology）：捕捉使用者的动作、视线与操作行为，用于任务表现分析与界面优化。；• 数据接口（Data Interface）：将设计过程与交互数据实时传输至分析平台，用于评估用户表现、空间舒适度与交互效率。

设计对象：• 空间（Architecture）：虚拟重建的航天栖息舱（Deep Space Habitat）与月球基地模块，用于测试空间动线、界面布局与人机协同。；• 交互系统（Interactive System）：用户通过 VR/MR 设备进行空间交互、任务操作与信息检索。；• 装置（Installation）：虚拟模拟设备（头显、控制界面、触觉反馈系统）与实体原型结合，形成可迭代测试的复合装置系统。

设计手法：• 模拟（Simulation）：利用虚拟现实与混合现实技术重建航天舱段、月球基地等内部布局，为宇航员训练和空间设计验证提供沉浸式原型测试环境。；• 具身化（Embodiment）：研究强调使用者（宇航员或工程师）在沉浸式环境中通过身体移动、操作、协作来感知空间结构与界面布局，从而评估设计的“身体适配度”。；• 迁移（Transference）：将地面建筑与人机工程的设计方法迁移到极端空间环境（轨道舱、月球基地）中，研究如何在虚拟与现实之间转译空间经验。；• 叙事化（Narrativization）：将虚拟舱段与任务流程构造成“任务场景”，用户在其中完成任务、测试交互、验证接口，形成“任务—体验—设计迭代”的叙事循环。

未来影响：可扩展为太空居住社区的共享、聚集、仪式与纪念等虚拟空间基础设施，不仅用于设计评审，更用于长期运营与社群仪式。

相关原理：虚拟/扩展现实；协作式虚拟环境；分布式参与

**NS-02｜Dataspace — A Reconfigurable Hybrid Reality Environment for Collaborative Information Analysis**

说明：该项目由 Marco Cavallo 等人提出，构建了一个融合实体（15 块高分辨率显示屏 + 机械臂）与 AR/VR 头显、移动设备的协作环境，用于团队共享、分析信息、可视化复杂数据。在这个环境中，参与者不拘泥于固定“房间”结构：显示屏可移动、虚拟元素可悬空、参与者可在“混合现实”空间自由部署界面。

启发：虽不是专为太空设计，但体现“场域可重构”“界面/空间非固定”，贴合太空中“舱室可移动、共享空间漂浮变化”的情境。；展现“用户驱动/动态聚集”的可能：参与者可依需求重构空间；在太空中，AI 或系统亦可根据社区活动生成共享场域。；作为“非地理性共享空间”的技术原型：在太空中地理意义弱化，此类混合现实空间可成为共同“广场”或“聚集地”的替代。

技术载体：• 感应技术（Sensing Technology）：Kinect 深度传感器、机器人臂扭矩传感器、语音识别麦克风阵列监测身体、手势、声音与屏幕／桌面互动。；• 沉浸式技术（Immersive Technology）：结合 AR／VR 头盔与物理屏幕环境，提供沉浸式数据分析与协作体验。；• 数据接口（Data Interface）：通过中间件 Merlin 与 MQTT 协议、机器人臂控制、投影／屏幕切换逻辑与云服务整合构建交互与设备控制接口。

设计对象：• 空间（Architecture）：房间级协作环境，包括可移动屏幕、投影桌、传感器、机器人臂、AR／VR 设备。；• 装置（Installation）：屏幕阵列、中央桌面投影、深度摄像头／Kinect 传感器、机器人臂构成的物理装置网络。；• 交互系统（Interactive System）：结合 AR／VR 头盔、移动设备、语音接口、手势识别与触屏／投影桌面，形成多模态互动系统。

设计手法：• 迁移（Transference）：将传统会议室／协作空间转化为高度可重构的混合现实环境，从桌面／屏幕界面迁移到“物理／虚拟融合”的立体空间。；• 模拟（Simulation）：整合 15 块由机器人臂可移动的高分辨率屏幕、中央投影桌、AR/VR 头盔与移动设备，模拟未来“协作室”中的身体／空间互动。；• 具身化（Embodiment）：参与者通过身体移动、手势、语音等与屏幕、投影、AR／VR 内容交互，使身体—空间—技术成为协作体验的一部分。；• 对比（Contrast）：在物理／虚拟、2D／3D、静态屏幕／可移动屏幕之间建立张力，凸显新型混合协作空间与传统桌面协作环境的差异。

未来影响：可迁移至太空/远程居住环境：分布式居住者进入共同虚拟场域，共享工作、学习、纪念；为在轨社区与跨地协作提供弹性、可重构的共享空间基建。

相关原理：虚拟-实体混合；空间重构；协作界面；沉浸交互

**NS-03｜Design Frameworks for Spatial Zone Agents in XRI Metaverse Smart Environments**

说明：这是一篇于 2024 年发表的研究论文，提出“XRI”（XR+IoT）环境中“空间区代理（Zone Agents）”的概念。作者提出在混合现实/智能环境中，将空间划分为“可感知、可交互、代理化”的区块，每个区块配置“代理”，作为用户/群体/环境之间的信息与行为中介，用于协调互动、数据流与场域规则。

启发：太空中地理分区失效，“空间区代理”提供基于活动/社群/情境的虚拟区块划分，贴合“非地理性共享空间”。；可设想太空社区中，AI 按需动态生成“纪念/学习/庆典”等功能场域，形成“流动纪念碑”“虚拟广场”等共享区块。；支持自适应治理：代理可根据传感数据与群体需求，调度界面、权限、交互规则与可视化。

技术载体：• 沉浸式技术（Immersive Technology）— 使用 XR（VR/AR/MR）设备，使用户进入混合现实空间并体验区代理互动。；• 感应技术（Sensing Technology）— 传感器检测用户位置、手势与环境变化，区代理基于这些输入作出响应。；• 数据接口（Data Interface）— 通过 IoT 通信（如 MQTT）、云端同步、用户行为与环境数据，支撑区代理的运行机制。

设计对象：• 交互系统（Interactive System）— Zone Agent 系统由用户、虚拟/物理代理、IoT 装置、XR 头显与交互界面构成互动生态。；• 空间（Architecture）— 智能空间被划分为多个“区（zones）”，如学习区、休闲区、会议区，每一区域具备特定代理逻辑、环境反馈与用户路径。；• 装置（Installation）— 混合现实头显、智能灯泡等 IoT 设备、虚拟代理模型、手势识别界面与环境传感器等安装于空间内部。

设计手法：• 迁移（Transference）— 将物理环境（用户房间、办公/社交空间）与虚拟／增强现实环境中的“空间区（Zone）”概念迁移结合，以区块（zones）作为智能代理（zone‑agents）在混合现实中运作。；• 模拟（Simulation）— 在 XRI（XR + IoT）智能环境中，空间区代理感知用户、环境与物联网设备，模拟“空间—代理—用户”的交互流程。；• 具身化（Embodiment）— 用户在混合现实环境中导航、移动并与区代理互动，身体动作、手势与位置成为代理识别与响应的触发机制。；• 叙事化（Narrativization）— 通过学习区、放松区、会议区等不同类型的 Zone Agents 与用户路径，形成“学习 → 放松 → 会议”的情境切换叙事。

未来影响：为太空共享空间提供系统化方法：以 AI/XR/IoT 构建、管理、演化场域区块；支持分布式居住的社群协作、仪式与公共活动。

相关原理：XR+IoT；区代理（Zone Agents）；智能环境；社群感知空间

出处：学术论文 arXiv: Design Frameworks for Spatial Zone Agents in XRI Metaverse Smart Environments (2024)；https://arxiv.org/abs/2401.11040?utm_source=chatgpt.com

标签：近未来；U院校；技术Technological, 社会Social；空间智能代理；XR 共享空间；非地理性场域；情境化区块

**NS-04｜Virtual Window for Deep Space Habitat（虚拟窗户在深空栖居舱）**

说明：由 NASA 研究团队提出，用于长时段/远地任务的栖居模块中，在物理窗户受限的情况下，通过 XR/投影系统在舱内生成“窗外景象”（如地球景色、自然场景、家人与纪念日影像等），以缓解隔离与单调带来的心理压力与认知负荷。

启发：直面太空居住“地理/自然连接缺失”的问题，以感知替代策略补偿。；虽偏向个体感知改善，但其“虚拟地景替代物理地景”的机制可扩展为多人共享的“虚拟公共广场/共享视窗”。；可在特定时间生成“共同观看”的仪式化窗口，作为非地理性的共享体验载体。

技术载体：• 光影控制（Lighting Control）：通过显示/投影/屏幕模拟窗外光线变化（昼夜、云层、外太空景观），影响舱内照明与氛围。；• 沉浸式技术（Immersive Technology）：以大屏或全景投影在封闭环境中创造“外部空间”感，缓解隔离感。；• 数据接口（Data Interface）：接入实时或预录视频流与环境数据（地球景观、外太空影像等），将数据映射至窗口显示，提供多样化视景。

设计对象：• 空间（Architecture）：深空栖息舱的内部结构与居住环境，虚拟窗口被作为空间系统的一部分，直接影响居住者的空间感知与心理状态。；• 装置（Installation）：由投影设备、显示屏、摄像头与计算节点组成的虚拟窗口装置，安装于栖息舱内以投射外部景观与情感内容。；• 交互系统（Interactive System）：乘员通过界面选择与切换窗口内容（自然景观、文化活动、家庭影像等），与虚拟窗口系统交互影响其空间体验。

设计手法：• 模拟（Simulation）：在深空栖息舱（DSH）内部通过“虚拟窗口”系统扩展乘员的感知空间，模拟外部环境、文化场景与家庭活动，以缓解长期隔离与狭小空间带来的心理负荷。；• 迁移（Transference）：将地球环境中“看窗外”“眺望自然”“观赏家庭视频”等体验迁移至封闭且远离地球的航天结构中，将地球的视觉与情感链接延伸到深空任务。；• 具身化（Embodiment）：强调使用者的身体—空间关系（面向投影、操作界面、切换场景等）以维护心理舒适与生理节律。；• 仪式化（Ritualization）：将“家庭视频通话”“生日祝福展示”“外部景观模拟”等具有仪式意义的活动纳入虚拟窗口的使用场景，将日常仪式结构引入太空居住空间。

未来影响：在多人舱、轨道群或月面基地中，发展为多人同步观看与互动的“虚拟外景+共处感”场域，用于情绪调节、团队凝聚、纪念活动与科普教育；与AI调度结合可按任务节律与社群需求自适应生成内容与交互规则。

相关原理：XR/投影视觉替代；心理/认知设计；共享观看体验

出处：学术论文 “Designing for Virtual Windows in a Deep Space Habitat”；https://spacearchitect.org/pubs/AIAA-2013-3529.pdf?utm_source=chatgpt.com

**NS-05｜Space Sharing System for Collaborative Mixed Reality**

说明：Meta / Unity 平台推出的 XR/混合现实“共享空间”技术。Shared Spaces 允许异地用户通过 XR 设备进入同一虚拟空间，内容/显示实时同步，用户可看到相同物件、位置与互动。ShapesXR 总结了四种共享方式：虚拟空间+行走/移动、以内容为锚点、虫洞/门户连接、数字孪生远程移动。

启发：非地理场景下的“多用户/异地/同一空间”能力，契合太空中需要共享场域的需求。；“内容为锚点”策略可将分布在不同舱室/轨道/站点的居民，通过AI驱动的虚拟共享广场汇聚到同一场域。；技术可支撑“任何舱室皆可进入”“分布式聚集”“动态场景生成”。

技术载体：• 沉浸式技术（Immersive Technology）：使用 Meta Quest MR 头显，融合虚拟图层与真实环境，实现空间共存与多用户沉浸式交互。；• 感应技术（Sensing Technology）：整合空间映射、深度感知、手势追踪、头部追踪与摄像机融合以实时检测环境。；• 数据接口（Data Interface）：依托 Shared Anchors API、Cloud Spatial Data 与 Network Sync 模块，实现空间数据的云端存储、识别与多用户同步。；• 物联网／智能空间（IoT / Smart Environment）：可与真实空间传感器与边缘计算节点协同，实现跨设备的空间数据共享与动态调整。

设计对象：• 交互系统（Interactive System）：Space Sharing API 允许多个设备（Meta Quest 3、Quest Pro 等）共享同一虚拟空间坐标，实现多人在 MR 中协作、游戏或共同创作。；• 空间（Architecture）：将虚拟对象锚定在用户物理房间内，实现“真实建筑”与“虚拟建筑层”的空间重叠。；• 数字平台／网络空间（Digital Platform / Online Space）：Horizon OS 及其 Unity SDK 作为开发与交互平台，使不同用户在网络连接下实时加入同一场景。

设计手法：• 迁移（Transference）：将现实世界的空间共享与社交行为迁移至混合现实（MR）环境，使多位用户在各自物理空间中“共处”于同一个虚拟叠加场景。；• 具身化（Embodiment）：用户通过 Meta Quest 设备进入共享空间，以身体位置、手势与视角实现实时交互与共在，形成“虚拟共体”体验。；• 模拟（Simulation）：在虚拟环境中重建每个用户的空间布局（房间边界、物体位置、平面检测结果），模拟真实空间交融。；• 生成（Generative）：通过 Meta Spatial Anchors API 与 Shared Spatial Data 系统，实时生成可被多用户共享和动态更新的空间坐标系。；• 叙事化（Narrativization）（隐性）：用户在共享 MR 空间中共同完成任务、表演或创造内容，使多用户互动构成共同故事线。

未来影响：为太空居民的虚拟聚集、仪式、学习、庆祝等提供平台基础，并可与AI场景编排结合，按任务节律与社群需求动态生成共享空间。

相关原理：XR/混合现实；多用户同步场域；内容锚点共享；数字孪生

**NS-06｜SpaceBlender — 生成式3D场景融合的协作虚拟空间**

说明：该研究提出 SpaceBlender 系统，使用生成式 AI 将用户所在的物理环境（2D 图片/场景）转化为富语境的 3D 虚拟空间，并支持多用户在该虚拟空间中协作。通过深度估计（depth estimation）、网格对齐（mesh alignment）、扩散式空间补全（diffusion-based space completion）等算法，把多方提供的场景融合为统一的协作虚拟场域，并用于 VR 协作任务。

启发：体现“不同物理空间/不同用户位置 → 同一虚拟场域”的核心特征，贴合“任何舱室可进入”“分布式居民实时聚合”。；其自动生成机制可视为“AI 驱动共享场域”的技术原型，依据集体活动自动生成与适配场景。；太空情境下：各舱室/模块提供影像或传感数据，AI 融合生成开放/共享的“虚拟广场/庆典场域”，属于非地理性共享空间。

技术载体：• 沉浸式技术（Immersive Technology）：使用 VR 头戴设备／手部控制器／空间追踪系统，让用户进入并操作该 3D 环境。；• 感应技术（Sensing Technology）：捕捉用户现实空间中的图片，进行深度估计、网格对齐、平面识别等以生成环境。；• 生成（Generative）／数据接口（Data Interface）：采用扩散模型（diffusion-based）、ControlNet、文本提示与几何先验生成场景；以用户图片、行为与环境数据作为接口输入。

说明：KAIST 团队以 Research through Design 的三阶段流程（参与式设计：遗愿者/悼念者共同参与 → VR 纪念空间开发 → 用户测试）探索“虚拟现实纪念空间（VR Memorial Space）”。研究指出：虚拟纪念空间可超越物理/地理限制，为不同地域与时间的人提供共同的纪念/仪式场所；多方利益相关者（Owner/Visitor）共同共创空间内容与形式；设计要素涵盖“想象性互动”（Imaginative Interaction Transcending Reality）与“空间听觉刺激”（Spatial Auditory Stimuli）等，以强化仪式与情感体验。
