Case Library 4: Non-geographic Shared Space

This case library focuses on the theme of "non-geographic shared space": when space loses its geographic meaning, how can a "sense of the public" and "consensus" be established? It collects cases concerned with directions such as "virtual space," "digital social interaction," "empathic systems," and "shared memory," providing references for the design of non-geographic public spaces and consensus systems.

**NS-01｜Thales Alenia Space "GATEWAY4U" Collaborative Virtual Environment**

Description: Developed by Thales Alenia Space, this project targets the design phase of space habitat modules by building a collaborative virtual environment in which engineers and astronauts can jointly experience and evaluate the architecture, layout, and functions of the module in XR (extended reality). Research shows that participants located in different places can "enter" the same virtual module to conduct discussion, design adjustment, and feedback.

Insight: Embodies "non-geographic shared space": multiple people enter the same virtual module from their respective locations. ; Supports "dynamic generation/adjustment": layout and functions can be changed in real time to simulate different scenarios. ; Provides a spatial model for XR collaboration and consensus formation for in-orbit, mobile, and position-changing modules.

Technology: • Immersive Technology: uses VR/MR headsets, spatial positioning and tracking, and real-time rendering technology to reconstruct the spaceflight environment. ; • Sensing Technology: captures the user's movements, gaze, and operating behavior, used for task performance analysis and interface optimization. ; • Data Interface: transmits design process and interaction data in real time to an analytics platform, used to evaluate user performance, spatial comfort, and interaction efficiency.

Design Object: • Architecture: virtually reconstructed Deep Space Habitat and lunar base modules, used to test spatial circulation, interface layout, and human-machine collaboration. ; • Interactive System: users perform spatial interaction, task operation, and information retrieval through VR/MR devices. ; • Installation: virtual simulation equipment (headsets, control interfaces, haptic feedback systems) combined with physical prototypes, forming a composite installation system for iterative testing.

Design Method: • Simulation: uses virtual reality and mixed reality technology to reconstruct the interior layouts of space habitat segments, lunar bases, and so on, providing an immersive prototype testing environment for astronaut training and spatial design validation. ; • Embodiment: the research emphasizes that the user (astronaut or engineer) perceives spatial structure and interface layout through bodily movement, operation, and collaboration in the immersive environment, thereby evaluating the "bodily fit" of the design. ; • Transference: transfers the design methods of terrestrial architecture and ergonomics into extreme space environments (orbital modules, lunar bases), studying how to translate spatial experience between the virtual and the real. ; • Narrativization: structures the virtual segments and task flows into "task scenarios" in which users complete tasks, test interactions, and validate interfaces, forming a "task–experience–design iteration" narrative loop.

**NS-02｜Dataspace — A Reconfigurable Hybrid Reality Environment for Collaborative Information Analysis**

Description: Proposed by Marco Cavallo and colleagues, this project builds a collaborative environment that fuses physical elements (15 high-resolution displays + robotic arm) with AR/VR headsets and mobile devices, used for teams to share, analyze information, and visualize complex data. In this environment, participants are not confined to a fixed "room" structure: displays can move, virtual elements can float, and participants can freely deploy interfaces in "mixed reality" space.

Insight: Although not designed specifically for space, it embodies "reconfigurable fields" and "non-fixed interfaces/spaces," fitting the scenario in space of "movable modules, shared spaces floating and changing." ; Demonstrates the possibility of "user-driven/dynamic gathering": participants can reconfigure space on demand; in space, AI or systems can likewise generate shared fields according to community activities. ; As a technical prototype of "non-geographic shared space": where geographic meaning is weakened in space, such mixed reality spaces can become a substitute for a common "plaza" or "gathering place."

Technology: • Sensing Technology: Kinect depth sensors, robotic arm torque sensors, and a voice-recognition microphone array monitor body, gestures, sound, and screen/tabletop interaction. ; • Immersive Technology: combines AR/VR headsets with a physical screen environment to provide an immersive data-analysis and collaboration experience. ; • Data Interface: builds interaction and device-control interfaces by integrating the middleware Merlin with the MQTT protocol, robotic arm control, projection/screen switching logic, and cloud services.

Design Object: • Architecture: a room-scale collaborative environment, including movable screens, a projection table, sensors, a robotic arm, and AR/VR devices. ; • Installation: a physical installation network composed of a screen array, central tabletop projection, depth cameras/Kinect sensors, and a robotic arm. ; • Interactive System: combines AR/VR headsets, mobile devices, voice interfaces, gesture recognition, and touchscreens/projection tabletops to form a multimodal interaction system.

Future Impact: Can be migrated to space/remote habitation environments: distributed inhabitants enter a common virtual field to share work, learning, and commemoration; provides flexible, reconfigurable shared-space infrastructure for in-orbit communities and cross-location collaboration.

Design Object: • Interactive System — the Zone Agent system forms an interactive ecosystem composed of users, virtual/physical agents, IoT devices, XR headsets, and interaction interfaces. ; • Architecture — the smart space is divided into multiple "zones," such as learning zone, leisure zone, and meeting zone, each with specific agent logic, environmental feedback, and user paths. ; • Installation — mixed reality headsets, IoT devices such as smart bulbs, virtual agent models, gesture-recognition interfaces, and environmental sensors are installed within the space.

Future Impact: Provides a systematic method for space shared spaces: builds, manages, and evolves field zones using AI/XR/IoT; supports community collaboration, ritual, and public activities for distributed habitation.
