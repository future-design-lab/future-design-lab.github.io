# Project Delivery Checker

Use at the end of a Future Design project.

Check:
1. Method logic: signal, STEEP, challenge, interpretation, provotyping, headline, backcasting, proposal.
2. Presentation richness: text explanation, visual assets, experience/process representation, and optional video or prototype.
3. File organization: project folder, asset index, prompt log, source files, and final exports.
4. Submission readiness: all final files are in `08_delivery/final-submission/`.

Output:

```text
## Delivery Check
Complete method artifacts:
Final presentation files:
Visual assets:
Prototype / video / diagrams:
Local archive status:
Missing items:
Recommended final improvements:
Submission folder:
```
